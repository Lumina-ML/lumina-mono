# Lumina

> Open-source MLOps platform. Rebased from WandB, built for self-hosting.

## Status

MVP V1 in progress. Current end-to-end flow works:

```bash
# Terminal 1: start backend + database
docker compose up

# Terminal 2: log metrics from Python
LUMINA_API_URL=http://localhost:8000 python -c "
import lumina
lumina.init(project='demo', name='exp-001', config={'lr': 0.01})
lumina.log({'loss': 0.9, 'acc': 0.1}, step=0)
lumina.finish()
"

# Open dashboard
open http://localhost:3000
```

## Architecture

- `apps/server` - Fastify backend API (Run + Metric modules)
- `apps/dashboard` - Web dashboard (placeholder HTML, Vue 3 in Week 5)
- `packages/shared` - Shared TypeScript types and schemas
- `packages/lumina-ui` - Internal UI component library
- `python/lumina` - Python SDK, rebased from WandB

## Quick Start

### Prerequisites

- Docker + Docker Compose
- Node.js 20 + pnpm 9
- Python 3.9+

### One-command start

```bash
docker compose up
```

Services:

| Service | Port | Description |
|---------|------|-------------|
| Dashboard | 3000 | Web UI |
| Server API | 8000 | REST API |
| Postgres | 15432 | Database (mapped to avoid local Postgres conflict) |

### Development

```bash
# Install monorepo dependencies
pnpm install

# Start Postgres
docker compose up postgres -d

# Run migrations (Postgres must be running)
pnpm --filter @lumina/server db:migrate

# Start backend
pnpm --filter @lumina/server dev

# Install Python SDK
cd python/lumina && pip install -e .
```

## SDK Usage

```python
import lumina

lumina.init(project="demo", name="exp-001", config={"lr": 0.01})
for i in range(10):
    lumina.log({"loss": 1.0 / (i + 1), "acc": i / 10.0}, step=i)
    lumina.log_system({"cpu": 10 + i, "memory": 100 + i * 10}, step=i)
    lumina.log_line(f"completed step {i}", level="INFO", step=i)

lumina.add_tag("baseline", color="#00ff00")
lumina.finish()
```

When `LUMINA_API_URL` is set, the SDK uses the Lumina backend. Otherwise it falls back to WandB-compatible behavior.

See [`examples/basic_experiment.py`](./examples/basic_experiment.py) for a full example.

## API

### Projects

- `POST /api/v1/projects` - Create a project
- `GET /api/v1/projects` - List projects
- `GET /api/v1/projects/:id` - Get a project
- `PATCH /api/v1/projects/:id` - Update a project
- `DELETE /api/v1/projects/:id` - Delete a project

### Runs

- `POST /api/v1/runs` - Create a run
- `GET /api/v1/runs` - List runs (filter by `project`, `status`, `createdAfter`, `createdBefore`)
- `GET /api/v1/runs/:runId` - Get a run
- `PATCH /api/v1/runs/:runId` - Update run status
- `DELETE /api/v1/runs/:runId` - Delete a run

### Metrics & Logs

- `POST /api/v1/runs/:runId/metrics` - Log metrics
- `GET /api/v1/runs/:runId/metrics` - Get metrics
- `POST /api/v1/runs/:runId/system-metrics` - Log system metrics
- `GET /api/v1/runs/:runId/system-metrics` - Get system metrics
- `POST /api/v1/runs/:runId/logs` - Log lines
- `GET /api/v1/runs/:runId/logs` - Get log lines

### Tags

- `POST /api/v1/projects/:projectId/tags` - Create a tag
- `GET /api/v1/projects/:projectId/tags` - List project tags
- `POST /api/v1/runs/:runId/tags` - Attach a tag to a run (by `tagId` or `name`)
- `GET /api/v1/runs/:runId/tags` - List run tags
- `DELETE /api/v1/runs/:runId/tags/:tagId` - Detach a tag from a run

### Artifacts

- `POST /api/v1/projects/:projectId/artifacts` - Create an artifact
- `GET /api/v1/projects/:projectId/artifacts` - List artifacts
- `GET /api/v1/artifacts/:artifactId` - Get an artifact
- `POST /api/v1/artifacts/:artifactId/versions` - Create a version
- `GET /api/v1/artifacts/:artifactId/versions` - List versions
- `GET /api/v1/versions/:versionId` - Get a version
- `PATCH /api/v1/versions/:versionId` - Update version aliases/state
- `POST /api/v1/versions/:versionId/files` - Register a file and get an upload URL

### Sweeps

- `POST /api/v1/projects/:projectId/sweeps` - Create a sweep
- `GET /api/v1/projects/:projectId/sweeps` - List sweeps
- `GET /api/v1/sweeps/:sweepId` - Get a sweep (includes runs)
- `PATCH /api/v1/sweeps/:sweepId` - Update sweep state

Run IDs are globally-unique UUID v7 values generated by the server, stable across regions and projects.

### Model Registry

- `POST /api/v1/projects/:projectId/registry-models` - Create a registered model
- `GET /api/v1/projects/:projectId/registry-models` - List registered models
- `GET /api/v1/registry-models/:modelId` - Get a registered model
- `POST /api/v1/registry-models/:modelId/versions` - Create a model version
- `GET /api/v1/registry-models/:modelId/versions` - List model versions
- `GET /api/v1/registry-model-versions/:versionId` - Get a model version
- `PATCH /api/v1/registry-model-versions/:versionId` - Update version aliases/metadata

### Evaluations

- `POST /api/v1/projects/:projectId/evaluations` - Create an evaluation
- `GET /api/v1/projects/:projectId/evaluations` - List evaluations
- `GET /api/v1/evaluations/:evaluationId` - Get an evaluation
- `PATCH /api/v1/evaluations/:evaluationId` - Update evaluation status/summary
- `POST /api/v1/evaluations/:evaluationId/results` - Add an evaluation result

### Traces & Spans

- `POST /api/v1/projects/:projectId/traces` - Create a trace
- `GET /api/v1/projects/:projectId/traces` - List traces
- `GET /api/v1/traces/:traceId` - Get a trace (includes spans)
- `PATCH /api/v1/traces/:traceId` - Update trace status/latency
- `POST /api/v1/traces/:traceId/spans` - Create a span
- `GET /api/v1/spans/:spanId` - Get a span
- `PATCH /api/v1/spans/:spanId` - Update span status/output

### Reports

- `POST /api/v1/projects/:projectId/reports` - Create a report
- `GET /api/v1/projects/:projectId/reports` - List reports
- `GET /api/v1/reports/:reportId` - Get a report
- `PATCH /api/v1/reports/:reportId` - Update a report
- `DELETE /api/v1/reports/:reportId` - Delete a report

### Tables & Media

- `POST /api/v1/projects/:projectId/run-media` - Register run media
- `GET /api/v1/projects/:projectId/run-media` - List run media
- `GET /api/v1/run-media/:runMediaId` - Get run media

## Artifacts

Lumina supports artifact upload and versioning with S3-compatible storage. Local development uses MinIO via Docker Compose.

```python
import lumina

artifact = lumina.LuminaArtifact(
    name="my-model",
    type="model",
    description="Trained model checkpoint",
)
artifact.add_file("path/to/model.pt")
artifact.save(project="demo", version="v1")

# Download it back
downloaded = lumina.use_lumina_artifact(
    "my-model",
    project="demo",
    alias="latest",
    download_dir="./downloads",
)
print(downloaded["downloaded"])
```

See [`examples/artifact_experiment.py`](./examples/artifact_experiment.py) for a full example.

## Sweeps

Lumina supports hyperparameter sweeps with `random`, `grid`, and a heuristic `bayes` scheduler.

```python
import lumina


def train(params):
    lr = params["lr"]
    epochs = params["epochs"]
    loss = 1.0 / (lr * 100) + epochs * 0.01
    return {"loss": loss}


config = {
    "method": "random",
    "parameters": {
        "lr": {"min": 0.001, "max": 0.1, "distribution": "log_uniform"},
        "epochs": {"values": [5, 10, 20]},
    },
    "metric": {"name": "loss", "goal": "minimize"},
}

sweep = lumina.sweep(config, project="demo", name="lr-sweep")
lumina.agent(sweep["id"], function=train, count=5, project="demo")
```

See [`examples/sweep_experiment.py`](./examples/sweep_experiment.py) for a full example.

## Model Registry

Lumina provides a Model Registry built on top of Artifacts. `log_model` uploads a model and registers a version; `use_model` downloads a version by alias.

```python
import lumina

# Upload and register a model version
lumina.log_model(
    path="path/to/model.pt",
    name="demo-classifier",
    aliases=["prod"],
    metadata={"framework": "pytorch", "epochs": 10},
    project="demo",
)

# Download the latest version
lumina.log_model(
    path="path/to/model-v2.pt",
    name="demo-classifier",
    aliases=["latest"],
    project="demo",
)

downloaded = lumina.use_model(
    name="demo-classifier",
    alias="latest",
    project="demo",
    download_dir="./downloads",
)
print(downloaded["downloaded"])
```

See [`examples/model_registry_experiment.py`](./examples/model_registry_experiment.py) for a full example.

## Evaluations

Lumina supports model evaluations that link a Run, a dataset ArtifactVersion, and a model ArtifactVersion.

```python
import lumina

# Create an evaluation referencing dataset/model artifacts by name
evaluation = lumina.init_eval(
    name="demo-eval",
    dataset="demo-dataset",
    model="demo-classifier",
    project="demo",
)

# Log evaluation metrics
lumina.log_eval_result("accuracy", 0.92)
lumina.log_eval_result("f1", 0.91)

# Mark evaluation as completed
lumina.finish_eval("completed")
```

## Traces & Spans

Lumina supports OpenTelemetry-style traces and spans for LLM / Agent observability.

```python
import lumina

with lumina.trace("chat-completion", project="demo", metadata={"model": "gpt-4"}) as trace:
    with lumina.span("retrieve-context", kind="retriever") as span:
        pass

    with lumina.span("llm-call", kind="llm", input_data={"messages": []}) as span:
        pass

    with lumina.span("tool-call", kind="tool") as span:
        pass
```

Span nesting is automatic inside a trace context. You can also use the manual APIs `start_trace()` / `finish_trace()` and `start_span()` / `finish_span()`.

See [`examples/trace_experiment.py`](./examples/trace_experiment.py) for a full example.

## Reports

Lumina supports rich reports composed of blocks (text, metrics, run galleries, etc.).

```python
import lumina

report = lumina.LuminaReport(title="Demo Report", project="demo")
report.add_text("Summary of the experiment.")
report.add_metric("final_loss", 0.2)
report.add_metric("final_acc", 0.8)
report.add_run_gallery(["run-id-1", "run-id-2"])

saved = report.save()
print(saved["id"])
```

## Tables & Media

Lumina supports logging complex data types (tables, images, videos, audio, Plotly charts) as RunMedia artifacts. Media values passed to `lumina.log()` are automatically detected and uploaded.

```python
import lumina

lumina.init(project="demo", name="media-run")

# Scalar metrics work as before
lumina.log({"loss": 0.2, "acc": 0.85}, step=0)

# Tables are automatically detected and stored as CSV artifacts
table = lumina.LuminaTable(columns=["epoch", "loss", "acc"])
table.add_row([0, 0.9, 0.1])
table.add_row([1, 0.5, 0.6])
table.add_row([2, 0.2, 0.85])
lumina.log({"metrics_table": table}, step=0)

# Explicit media logging
lumina.log_media("notes", {"text": "experiment notes"}, type="file")

lumina.finish()
```

See [`examples/media_experiment.py`](./examples/media_experiment.py) for a full example.

## Storage configuration

The server supports two storage modes via `STORAGE_TYPE`:

- `local` - Files stored on disk; upload/download goes through the server.
- `s3` - S3-compatible storage (MinIO, AWS S3, etc.) using presigned URLs.

Environment variables are documented in [`apps/server/.env.example`](./apps/server/.env.example).
